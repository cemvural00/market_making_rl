# Results Directory

This directory contains all experimental results, evaluation outputs, and generated reports.

## Directory Structure

```
results/
├── {EnvName}/              # Environment-specific results
│   └── {AgentName}/        # Agent-specific results
│       ├── metrics.json    # Performance metrics
│       ├── pnls.npy        # PnL array (one value per episode)
│       ├── inventory.npy   # Final inventory array
│       └── pnl_distribution.png  # Basic PnL distribution plot
│
├── figures/                # All visualization figures
│   ├── heatmap_*.png       # Metric heatmaps
│   ├── category_comparison_*.png  # Category comparisons
│   ├── agent_comparison_*.png     # Individual agent comparisons
│   ├── risk_return_*.png   # Risk-return plots
│   ├── ranking_*.png       # Agent rankings
│   ├── agent_risk_return_*.png    # Per-agent risk-return profiles
│   └── pnl_dist_*.png      # PnL distribution plots
│
├── logs/                   # Training logs (if any)
│
├── EVALUATION_REPORT.md    # Detailed evaluation report
├── APPENDIX.md             # Matrix format appendix tables
├── VISUALIZATION_REPORT.md # Comprehensive visualization report
├── VISUALIZATION_REPORT.html  # Interactive HTML report
├── RESULTS_AND_CONCLUSIONS.md  # Results and conclusions summary
├── AGENT_RISK_RETURN_PROFILES.md  # Risk-return profiles per agent
├── PNL_DISTRIBUTIONS.md    # PnL distribution plots
│
└── comparison_table.csv    # Aggregated comparison table
```

## Result Files

### Per Experiment Results

Each agent-environment combination creates a directory with:

#### `metrics.json`
Performance metrics in JSON format:
```json
{
    "mean": 2.4567,
    "std": 18.2341,
    "sharpe": 0.1347,
    "var_95": -35.1234,
    "var_99": -52.3456,
    "es_95": -45.6789,
    "es_99": -68.9012,
    "avg_inventory": 1.2345
}
```

**Metrics:**
- `mean`: Mean PnL across episodes
- `std`: Standard deviation of PnL
- `sharpe`: Sharpe ratio (mean / std)
- `var_95`: Value at Risk at 95% confidence (5th percentile)
- `var_99`: Value at Risk at 99% confidence (1st percentile)
- `es_95`: Expected Shortfall at 95% confidence
- `es_99`: Expected Shortfall at 99% confidence
- `avg_inventory`: Average final inventory position

#### `pnls.npy`
NumPy array containing final PnL value for each evaluation episode.
- Shape: `(n_episodes,)`
- Type: `float64`
- Example: `[1.5, 2.3, -0.5, 3.1, ...]`

Used for:
- Statistical analysis
- Distribution plots
- Confidence interval calculations

#### `inventory.npy`
NumPy array containing final inventory position for each evaluation episode.
- Shape: `(n_episodes,)`
- Type: `float64`
- Example: `[2.0, -1.5, 0.0, 3.5, ...]`

Used for:
- Inventory management analysis
- Risk assessment

#### `pnl_distribution.png`
Basic histogram of PnL distribution (automatically generated).

## Data Loading

### Python API

Load metrics:
```python
from scripts.aggregate_results import load_all_metrics

metrics_dict = load_all_metrics("results")
# Returns: {env_name: {agent_name: metrics_dict}}
```

Load PnL arrays:
```python
import numpy as np

pnls = np.load("results/ABMVanillaEnv/PPOAgent/pnls.npy")
```

Load all data with confidence intervals:
```python
from scripts.create_visualization_report import load_all_data

metrics_dict, pnl_data_dict, ci_dict = load_all_data("results")
```

### Pandas DataFrame

Convert to DataFrame for analysis:
```python
import pandas as pd
from scripts.aggregate_results import load_all_metrics

metrics_dict = load_all_metrics("results")
rows = []
for env_name, agents in metrics_dict.items():
    for agent_name, metrics in agents.items():
        row = {"Environment": env_name, "Agent": agent_name, **metrics}
        rows.append(row)

df = pd.DataFrame(rows)
```

## Regenerating Reports

To regenerate all reports:

```bash
# Generate all reports
python scripts/create_evaluation_report.py
python scripts/create_appendix.py
python scripts/create_visualization_report.py
python scripts/create_results_summary.py
python scripts/create_agent_risk_return_plots.py
python scripts/create_pnl_distributions.py
```

Reports are regenerated from existing result files, so you don't need to re-run experiments.

## File Sizes

Typical file sizes:
- `metrics.json`: ~1-2 KB per experiment
- `pnls.npy`: ~10-50 KB per experiment (depends on n_episodes)
- `inventory.npy`: ~10-50 KB per experiment
- `pnl_distribution.png`: ~50-100 KB per experiment
- Report markdown files: 100 KB - 1 MB
- Visualization figures: 100 KB - 500 KB each

Total for full experiment suite (12 envs × 17 agents):
- Result files: ~5-10 MB
- Figures: ~50-100 MB
- Reports: ~5-10 MB

## Best Practices

1. **Keep raw data**: Always keep `pnls.npy` and `inventory.npy` for future analysis
2. **Regenerate reports**: Reports can be regenerated anytime from existing results
3. **Version control**: Consider versioning important result directories
4. **Backup**: Backup `results/` directory before major changes
5. **Organization**: Results are automatically organized by environment and agent

## Troubleshooting

### Missing Results
If results are missing for an agent-environment combination:
- Check if the experiment completed successfully
- Verify the agent was evaluated (not just trained)
- Run `scripts/compare_all_agents.py` to fill gaps

### Corrupted Files
If JSON or NumPy files are corrupted:
- Re-run the specific experiment
- Use `scripts/compare_all_agents.py --no-skip` to regenerate

### Report Generation Errors
If report generation fails:
- Ensure all required result files exist
- Check that all agents have results for all environments
- Verify file permissions
